# Прогноз потребления и выручки (вкладка «Статистика → Прогноз»)

## Источники данных

Данные берутся из локальной SQLite БД:

- `invoices(date, total, document_type, ...)`
- `invoice_items(product_id, quantity, total, unit_weight_g, ...)`
- `products(code, internal_code, category, weight, ...)`

### Вес единицы (`unit_weight_g`)

- Для новых инвойсов `unit_weight_g` пишется в `invoice_items` при создании.
- Для старых записей, где `unit_weight_g` = NULL, используется fallback `products.weight` (актуальное значение).

## Базовые метрики

По каждой транзакции считаем:

- **Единицы** = `quantity`
- **Вес (кг)** = `quantity × unit_weight_g / 1000`
- **Выручка (RSD)** = `invoice_items.total`

Агрегации выполняются **по месяцам** (формат `YYYY-MM`). Месяцы с нулевыми продажами включаются как 0.

## Стабильность SKU

Стабильность — доля месяцев, в которых товар продавался:

`stability = sold_months / total_months`

Используется для:

- отбора Top‑10 (только SKU со стабильностью ≥ 0.5)
- маркировки ненадёжных SKU (🔴 если < 0.25)

## Режимы прогноза

### 1) Без роста

Коэффициент роста = 0.

### 2) Предустановленные / ручные проценты

Формула для горизонта \(N\):

`forecast_total = mean_monthly × N × (1 + growth)`

Где `growth` задаётся в UI (по умолчанию 15/25/35% для 3/6/12 месяцев).

### 3) Авто‑тренд (линейная регрессия)

Берём последние 12 месяцев (или меньше, если данных меньше) и строим регрессию:

`y(t) = a·t + b`

Показываем **R²**; если \(R² < 0.3\) — выводится предупреждение о слабом тренде.

Если истории меньше 6 месяцев — авто‑режим считается ненадёжным (UI показывает предупреждение).

## Интерпретация R²

- **R² ≈ 1.0**: линейный тренд хорошо объясняет данные
- **R² < 0.3**: тренд слабый, прогноз по тренду может быть неточным → рекомендуется ручной режим
